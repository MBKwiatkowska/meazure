00193 /// The star of the show, this call performs XML validation for the expat
00194 /// parser. An application instantiates a Validator object and ties it
00195 /// into the expat parser via the parser's handler functions.
00196 ///
00197 /// The validator constructs a DFA for each element's content model using
00198 /// the venerable algorithm 3.5 from the Dragon book. Optimizations bypass
00199 /// the algorithm and directly construct the DFA for simple content models
00200 /// (e.g. singletons, pure choice, pure sequence).
00201 ///
00202 /// The validator is a C++ object with methods that you call from expat
00203 /// handlers. The method parameters are of the same types as those passed
00204 /// to the expat handlers.
00205 ///
00206 /// To use the validator instantiate an ev::Validator object then call the
00207 /// following methods from the corresponding expat handlers:
00208 ///
00209 /// <table align="center" cellspacing="0" cellpadding="3">
00210 ///     <tr><th>Validator Method</th>   <th>Call From expat Handler</th></tr>
00211 ///     <tr><td>AddElementDecl</td>     <td>XML_ElementDeclHandler</td></tr>
00212 ///     <tr><td>AddAttributeDecl</td>   <td>XML_AttlistDeclHandler</td></tr>
00213 ///     <tr><td>AddNotationDecl</td>    <td>XML_NotationDeclHandler</td></tr>
00214 ///     <tr><td>AddEntityDecl</td>      <td>XML_EntityDeclHandler</td></tr>
00215 ///     <tr><td>SetDocumentElement</td> <td>XML_StartDoctypeDeclHandler</td></tr>
00216 ///     <tr><td>StartElement</td>       <td>XML_StartElementHandler</td></tr>
00217 ///     <tr><td>EndElement</td>         <td>XML_EndElementHandler</td></tr>
00218 ///     <tr><td>CharacterData</td>      <td>XML_CharacterDataHandler</td></tr>
00219 /// </table>
00220 ///
00221 /// Encoding uses expat's XML_Char encoding based on whether XML_UNICODE is
00222 /// defined. exval defines the types ev::EVString and ev::EVOStream but these
00223 /// are just convenience wrappers around the appropriate STL string and stream
